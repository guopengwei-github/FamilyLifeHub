# Garmin健康数据看板重设计

## 概述

将FamilyLifeHub从通用家庭数据中心精简为专注于Garmin健康数据的看板应用。

## 目标

1. 移除工作数据模块，聚焦Garmin健康数据
2. 简化代码库，删除非必要代码
3. 优化数据展示，支持趋势追踪
4. 保留家庭成员功能，支持异常提醒

## 功能范围

### 保留功能

- 家庭成员管理
- Garmin OAuth连接与数据同步
- 健康数据展示（睡眠、活动、生理指标、压力）
- 趋势图表（7天/30天/自定义）
- 数据异常视觉提示

### 移除功能

- 工作数据模块（WorkMetric）
- Strava集成
- 工作相关UI组件

### 后期功能

- 大模型分析报告
- 异常推送通知
- 锻炼建议

## 数据模型

### 保留的模型

```
User
├── id, name, email, avatar
├── GarminConnection (1:1)
├── GarminActivity (1:N)
└── HealthMetric (1:N)
```

### HealthMetric 核心字段

| 类别 | 字段 |
|------|------|
| 睡眠 | sleep_hours, sleep_score, light_sleep_hours, deep_sleep_hours, rem_sleep_hours |
| 活动 | steps, distance_km, calories, exercise_minutes |
| 生理 | resting_heart_rate, spo2, respiration_rate |
| 状态 | stress_level, body_battery |

### 删除的模型

- WorkMetric 表
- StravaConnection 表
- ActivityMetric 表（Strava活动）

## 异常检测规则

视觉提示，用颜色/图标标记：

| 指标 | 正常（绿） | 警告（黄） | 异常（红） |
|------|------------|------------|------------|
| 睡眠时长 | 7-9小时 | 6-7或9-10 | <6或>10 |
| 静息心率 | 50-70 | 70-80 | >80 |
| 压力等级 | 0-40 | 40-60 | >60 |
| Body电量 | 50-100 | 25-50 | <25 |

## API 端点

### 保留

- `GET /api/v1/users` - 用户列表
- `GET /api/v1/dashboard/summary` - 摘要数据
- `GET /api/v1/dashboard/overview` - 今日概览
- `GET /api/v1/dashboard/trends` - 趋势数据
- `POST /api/v1/garmin/*` - Garmin连接与同步

### 删除

- `POST /api/v1/ingest/work` - 工作数据摄入
- `POST /api/v1/strava/*` - Strava相关端点

## 实现阶段

### 阶段一：清理（移除不需要的代码）

1. 删除 WorkMetric 模型和相关 schema
2. 删除 Strava 相关模型和服务
3. 删除工作数据 API 端点
4. 删除前端工作相关组件
5. 清理 dashboard 服务中的工作数据逻辑

### 阶段二：优化（改进现有功能）

1. 简化 Dashboard 布局
2. 添加时间范围切换（7天/30天/自定义）
3. 优化趋势图表展示
4. 添加异常数据视觉提示

### 阶段三：增强（家庭成员功能）

1. 家庭成员概览卡片
2. 成员间数据对比
3. 异常数据高亮显示
